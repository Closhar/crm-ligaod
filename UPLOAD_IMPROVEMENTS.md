# Улучшения системы загрузки изображений

## Решение проблемы с загрузкой больших файлов (413 Content Too Large)

### Проблема
При загрузке большого количества файлов (например, 45 файлов) одним запросом возникает ошибка 413 "Content Too Large", так как общий размер запроса превышает лимиты сервера.

### Решение
Реализована загрузка файлов по одному с отображением прогресса:

#### Фронтенд (Nuxt 3)
- **Загрузка по одному файлу**: Вместо отправки всех файлов одним запросом, файлы отправляются последовательно
- **Индикатор прогресса**: Показывает текущий файл, процент выполнения и общий прогресс
- **Возможность отмены**: Пользователь может отменить загрузку в любой момент
- **Детальная отчетность**: Показывает количество успешно загруженных файлов и ошибки

#### Серверная сторона (Laravel)
- **Увеличены лимиты**: В `api_routes.php` и `.htaccess` увеличены лимиты загрузки
- **Обработка одного файла**: Контроллер принимает один файл за раз
- **Валидация каждого файла**: Каждый файл валидируется отдельно
- **Обработка изображений**: Используются встроенные PHP функции GD для изменения размера и создания миниатюр

### Обработка изображений (без внешних библиотек)

Система использует встроенные PHP функции GD для обработки изображений:

#### Поддерживаемые форматы:
- JPEG (.jpg, .jpeg)
- PNG (.png) - с поддержкой прозрачности
- GIF (.gif)
- WebP (.webp)

#### Функции обработки:
- **Изменение размера**: Максимальная ширина 1600px с сохранением пропорций
- **Создание миниатюр**: Высота 50px с сохранением пропорций
- **Конвертация в JPEG**: Все изображения сохраняются в формате JPEG с качеством 80%
- **Сохранение прозрачности**: PNG изображения корректно обрабатываются

#### Используемые PHP функции:
```php
// Загрузка изображений
imagecreatefromjpeg() - для JPEG
imagecreatefrompng() - для PNG
imagecreatefromgif() - для GIF
imagecreatefromwebp() - для WebP

// Изменение размера
imagecreatetruecolor() - создание нового изображения
imagecopyresampled() - изменение размера с интерполяцией

// Сохранение
imagejpeg() - сохранение в JPEG формате

// Управление памятью
imagedestroy() - освобождение памяти
```

### Настройки лимитов

#### PHP настройки (api_routes.php)
```php
ini_set('upload_max_filesize', '100M');
ini_set('post_max_size', '100M');
ini_set('max_execution_time', 300);
ini_set('memory_limit', '512M');
```

#### Apache настройки (.htaccess)
```apache
php_value upload_max_filesize 100M
php_value post_max_size 100M
php_value max_execution_time 300
php_value memory_limit 512M
php_value max_input_time 300
```

### Преимущества нового подхода

1. **Надежность**: Загрузка по одному файлу исключает ошибки 413
2. **Отслеживание прогресса**: Пользователь видит, какой файл загружается
3. **Возможность отмены**: Можно остановить загрузку в любой момент
4. **Детальная отчетность**: Показывает успехи и ошибки для каждого файла
5. **Обработка ошибок**: Если один файл не загрузился, остальные продолжают загружаться
6. **Без внешних зависимостей**: Используются только встроенные PHP функции

### Использование

1. Выберите галерею из списка слева
2. Нажмите "Добавить изображения"
3. Выберите файлы (можно много)
4. Нажмите "Загрузить"
5. Следите за прогрессом загрузки
6. При необходимости нажмите "Отменить загрузку"

### Логирование

Все операции загрузки логируются в консоль браузера:
- Начало загрузки с количеством файлов
- Прогресс каждого файла
- Успешные загрузки (✅)
- Ошибки загрузки (❌)
- Итоговый отчет

### Возможные проблемы и решения

1. **Ошибка 413 все еще возникает**: Проверьте настройки веб-сервера (nginx/apache)
2. **Медленная загрузка**: Увеличьте паузу между запросами (сейчас 100ms)
3. **Превышение лимита времени**: Увеличьте `max_execution_time` в настройках PHP
4. **Ошибка GD extension**: Убедитесь, что PHP GD extension установлен и включен

### Требования к серверу

- PHP 7.4+ с включенным GD extension
- Достаточно памяти для обработки изображений (рекомендуется 512M+)
- Права на запись в папку storage/app/public

## Как это работает

### На фронтенде (Nuxt):
```javascript
// Создаем FormData для всех файлов
const formData = new FormData();
selectedFiles.value.forEach((file, index) => {
  formData.append('images[]', file);
});

// Отправляем одним запросом
const response = await $fetch('/api/v1/galleries/{id}/upload-image', {
  method: 'POST',
  body: formData
});
```

### На бэкенде (Laravel):
```php
// Получаем все файлы
$files = [];
if ($request->hasFile('image')) {
    $files[] = $request->file('image');
}
if ($request->hasFile('images')) {
    $files = array_merge($files, $request->file('images'));
}

// Обрабатываем по очереди
foreach ($files as $file) {
    // Валидация
    // Обработка изображения
    // Сохранение в БД
}
```
